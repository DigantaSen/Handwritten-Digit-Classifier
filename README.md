# MNIST Handwritten Digit Classifier

An end-to-end deep learning project that trains three neural network architectures on the MNIST dataset and exposes them through an interactive Streamlit web app.

🚀 **Live Demo:** [handwritten-digit-classifier-by-diganta.streamlit.app](https://handwritten-digit-classifier-by-diganta.streamlit.app/)

---

## Dataset

The MNIST dataset used in this project is sourced from Kaggle:
**[MNIST Deep Neural Network with Keras](https://www.kaggle.com/code/prashant111/mnist-deep-neural-network-with-keras)**

Download `mnist_train.csv` and `mnist_test.csv` and place them in the project root before running the notebook.

## Models

Three architectures are trained and compared side-by-side:

| Model | Architecture | Optimizer | Notes |
|-------|-------------|-----------|-------|
| **Perceptron** | Flatten → Dense(10, softmax) | SGD | Linear classifier baseline |
| **ANN** | Flatten → Dense(256) → Dense(128) → Dense(64) → Dense(32) → Dense(10, softmax) | Adam | Fully-connected deep network |
| **CNN** | Conv2D(32) → MaxPool → Conv2D(64) → MaxPool → Conv2D(128) → Flatten → Dense(128) → Dropout(0.5) → Dense(10, softmax) | Adam | Best accuracy; spatial feature extraction |

All models use `categorical_crossentropy` loss and are trained for up to 500 epochs with a batch size of 256. Input pixels are normalised to `[0, 1]`.

## Usage

### Step 1 — Train and save the models

Open and run all cells in **`project.ipynb`**. This will:

1. Load `mnist_train.csv` and `mnist_test.csv`
2. Preprocess and normalise the data
3. Train the Perceptron, ANN, and CNN
4. Save the trained models to the `models/` directory
5. Plot training curves, a validation accuracy comparison, sample predictions, and a CNN confusion matrix

> **Note:** Training the CNN for 500 epochs can take a while. A GPU is recommended (`tf.config.list_physical_devices('GPU')` is printed during training).

### Step 2 — Launch the web app

```bash
streamlit run app.py
```

The app opens at `http://localhost:8501`.

## Web App Features

### Input modes
- **Draw on canvas** — freehand draw a digit (0–9) on a 280×280 black canvas using `streamlit-drawable-canvas`
- **Upload image** — upload a PNG/JPG/BMP image of a digit

### Prediction
Click **🔮 Predict** to run the input through all three models simultaneously. For each model the app shows:
- The predicted digit (large display)
- Confidence percentage
- A probability bar chart (0–9) powered by Plotly

### Agreement banner
- If all three models agree → a green success banner
- If models disagree → an info banner showing each model's prediction and the majority-vote result

### Image preprocessing
All inputs (canvas or upload) are:
1. Converted to grayscale
2. Resized to 28×28 using Lanczos resampling
3. Normalised to `[0, 1]`
4. Reshaped to `(1, 28, 28)` for Perceptron/ANN or `(1, 28, 28, 1)` for CNN

## Notebook Walkthrough (`project.ipynb`)

| Section | What it does |
|---------|-------------|
| Imports | Loads NumPy, Pandas, Matplotlib, Seaborn, Scikit-learn, Keras |
| Data loading | Reads CSV files; separates features and labels |
| Preprocessing | Normalise to `[0,1]`, reshape for image/CNN formats, one-hot encode labels |
| Perceptron | Build → compile (SGD) → fit 100 epochs → evaluate |
| ANN | Build deep FC network → compile (Adam) → fit 100 epochs → evaluate |
| CNN | Build conv network → compile (Adam) → fit 500 epochs → evaluate |
| Save models | Saves all three `.h5` files to `models/` |
| Visualisation | Training/validation accuracy & loss curves per model |
| Comparison | Overlay validation accuracy for all three models |
| Sample predictions | Side-by-side visual comparison on random test images |
| Confusion matrix | CNN confusion matrix heatmap (Seaborn) |
| Accuracy bar chart | Final test accuracy comparison across all three models |
